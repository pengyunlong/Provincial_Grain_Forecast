# 省份聚类方法说明

生成日期：2026-05-26

本文用于说明 `Province_Clusting` 目录下四类省份聚类的技术方法、相同点、差异点、簇数量选择规则和主要参数设计。这里的“四种聚类方法”实际对应四个独立聚类任务：口粮消费聚类、饲料粮消费聚类、人口结构聚类、经济水平与价格指标聚类。

## 1. 总体结论

四类聚类的算法流程相同，但业务指标体系不同。

相同点：

1. 聚类对象均为 31 个省级行政区，不含港澳台。
2. 都先将年度面板数据整理为省份级特征表。
3. 都使用 `StandardScaler` 思路进行标准化，即按每个特征列做均值为 0、标准差为 1 的 z-score 转换。
4. 都在标准化后的特征空间中执行 KMeans 聚类。
5. 都在 `K=2,3,4,5,6` 范围内评估簇数量。
6. 都输出 `K值评估表.csv`、`省份聚类结果.csv`、聚类中心表、质量检查表和可视化图。

不同点：

1. 四类聚类使用的数据来源不同。
2. 口粮和饲料粮使用 2015-2024 年，人口和经济使用 2000-2024 年。
3. 四类聚类的特征字段不同，因此特征数量也不同。
4. 实际选出的默认 K 值不同：口粮 K=2，饲料粮 K=2，人口 K=4，经济 K=2。
5. 标签解释规则不同：口粮、饲料粮和经济主要按水平与趋势命名，人口按人口规模、老龄化程度和人口趋势命名。

## 2. 通用聚类流程

四类任务的脚本虽然分开存放，但都调用 `common/clustering_utils.py` 中的通用函数。核心流程如下：

```text
读取数据
清洗字段和年份
构造省份级特征
处理缺失值
标准化数值特征
评估 K=2-6
按规则选择默认 K
执行 KMeans
生成标签、结果表和聚类中心
输出 PCA 散点图、中心热图、省份数量图、排序图
输出质量检查表
```

PCA 仅用于二维散点图展示，不参与聚类建模。实际聚类使用的是标准化后的完整特征表。

## 3. 簇数量 K 的选择方法

四类任务统一在 `K=2` 到 `K=6` 之间评估，评估指标包括：

| 指标 | 解释 | 选择倾向 |
| :--- | :--- | :--- |
| `inertia` | 类内平方和，越小表示类内越紧凑 | 越小越好，但会随 K 增大自然下降 |
| `silhouette_score` | 轮廓系数，衡量类内紧凑和类间分离 | 越大越好 |
| `calinski_harabasz_score` | 类间离散与类内离散的比值 | 越大越好 |
| `davies_bouldin_score` | 类内散度与类间距离的比值 | 越小越好 |

默认 K 的程序选择规则为：

1. 先按 `silhouette_score` 降序排序。
2. 若轮廓系数相同或接近，再参考 `calinski_harabasz_score` 越大越好、`davies_bouldin_score` 越小越好。
3. 取最佳轮廓系数 `best_score`，将 `silhouette_score >= best_score - 0.03` 的 K 视为接近最优。
4. 如果接近最优的 K 中包含 3 或 4，则优先选 3 或 4 中轮廓系数更高者，保证分类结果便于解释和汇报。
5. 如果没有接近最优的 3 或 4，则直接选择综合排序第一的 K。

四类任务的实际 K 值如下：

| 聚类任务 | 默认 K | K 值选择说明 |
| :--- | ---: | :--- |
| 口粮消费聚类 | 2 | K=2 的轮廓系数最高，为 0.7867，明显优于其他 K，因此选择 2 类。 |
| 饲料粮消费聚类 | 2 | K=2 的轮廓系数最高，为 0.4255，因此选择 2 类。 |
| 人口结构聚类 | 4 | K=6 的轮廓系数最高，为 0.3030；K=4 的轮廓系数为 0.2886，处于 `best_score - 0.03` 的接近最优范围内。根据“优先选择 3 或 4 以便解释”的规则，最终选择 K=4。 |
| 经济水平与价格指标聚类 | 2 | K=2 的轮廓系数最高，为 0.3016，因此选择 2 类。 |

## 4. KMeans 参数设计

四类聚类的 KMeans 参数一致：

| 参数 | 取值 | 设计说明 |
| :--- | :--- | :--- |
| 聚类算法 | KMeans | 适合在标准化后的连续数值特征上进行硬聚类，结果便于生成聚类中心和标签。 |
| 距离度量 | 欧氏距离 | KMeans 使用平方欧氏距离最小化类内离差。 |
| `n_clusters` | 各任务选出的默认 K | 口粮 2、饲料粮 2、人口 4、经济 2。 |
| `random_state` | 42 | 固定随机种子，保证复跑结果稳定。 |
| `n_init` | 80 | 随机初始化 80 次，取 `inertia` 最小的结果，降低初始化偶然性。 |
| `max_iter` | 300 | 每次初始化最多迭代 300 轮，满足常规 KMeans 收敛需要。 |
| 空簇处理 | 以距离最近中心最远的样本重置空簇中心 | 防止某次迭代产生空簇导致算法中断。 |
| 标准化 | 每列减均值、除标准差；标准差接近 0 时置为 1 | 避免 GDP、人口、比例、价格指数等不同量纲直接支配距离。 |

## 6. 四类聚类的具体说明

### 6.1 口粮消费聚类

口粮聚类使用 `数据提取与可视化/口粮/table/省级口粮长表数据.csv`，筛选 2015-2024 年 31 省口粮消费数据。输入变量为 `口粮（千克/人）`，每个省构造 8 个水平、趋势和波动特征。

聚类结果选择 K=2，得到：

| 类别 | 标签 | 省份数量 |
| :--- | :--- | ---: |
| 0 | 高水平-下降型 | 1 |
| 1 | 低水平-稳定型 | 30 |

该任务中 K=2 的轮廓系数显著最高，说明存在一个口粮水平明显较高且趋势下降的省份，其余省份在口粮消费水平和趋势上更接近。

### 6.2 饲料粮消费聚类

饲料粮聚类使用 `数据提取与可视化/七类动物性消费/饲料粮/table/省级饲料粮长表数据.csv`，筛选 2015-2024 年 31 省饲料粮折算数据。输入变量为 `饲料粮折算值（千克/人）`，特征结构与口粮聚类相同。

聚类结果选择 K=2，得到：

| 类别 | 标签 | 省份数量 |
| :--- | :--- | ---: |
| 0 | 低水平-上升型 | 14 |
| 1 | 高水平-上升型 | 17 |

该任务中的两个簇主要体现饲料粮消费水平差异，两个类别的趋势均表现为上升型。

### 6.3 人口结构聚类

人口聚类使用 `宏观数据收集（CSV编码）/人口数据（省级）.csv`，筛选 2000-2024 年数据。基础字段包括人口规模、城乡人口、年龄结构、平均家庭规模和户数，并派生城镇化率、乡村人口占比、0-14 岁占比、15-64 岁占比、65 岁及以上占比。

人口聚类没有纳入标准差和变异系数，原因是人口结构任务更关注规模、结构和长期方向，使用 13 个基础及派生字段分别构造 6 个特征，共 78 个特征。

聚类结果选择 K=4，得到：

| 类别 | 标签 | 省份数量 |
| :--- | :--- | ---: |
| 0 | 中等人口规模-老龄化中-上升型 | 9 |
| 1 | 高人口规模-老龄化中-上升型 | 1 |
| 2 | 低人口规模-老龄化高-上升型 | 7 |
| 3 | 中等人口规模-老龄化低-上升型 | 14 |

人口聚类的轮廓系数最高值出现在 K=6，但 K=4 与最高值差距小于 0.03。为了避免人口任务拆分过细、提高汇报解释性，程序按统一规则选择 K=4。

### 6.4 经济水平与价格指标聚类

经济聚类使用 `宏观数据收集（CSV编码）/经济数据（省级）.csv`，筛选 2000-2024 年数据。使用字段包括地区生产总值、人均可支配收入总计、人均消费支出总计、总 CPI、食品 CPI 和粮食 CPI。

经济聚类明确排除分城乡字段，包括城镇居民、农村居民、城市、农村等口径，避免和总计口径混用。

聚类结果选择 K=2，得到：

| 类别 | 标签 | 省份数量 |
| :--- | :--- | ---: |
| 0 | 高经济水平-上升型 | 7 |
| 1 | 低经济水平-上升型 | 24 |

该任务中 K=2 的轮廓系数最高，说明经济指标更适合先划分为高经济水平和低经济水平两类。

## 7. 标签生成参数

口粮、饲料粮和经济聚类使用统一的水平与趋势标签规则：

1. 对每个簇计算核心水平指标的簇中心均值。
2. 使用簇中心的 33% 和 67% 分位点划分低水平、中等水平、高水平。
3. 对趋势指标计算簇中心均值。
4. 趋势斜率 `> 0.5` 标记为上升型，`< -0.5` 标记为下降型，介于两者之间标记为稳定型。

人口聚类使用定制标签规则：

1. 使用 `总人口_最新年值` 的簇中心 33% 和 67% 分位点划分低人口规模、中等人口规模、高人口规模。
2. 使用 `65岁及以上占比_最新年值` 的簇中心 33% 和 67% 分位点划分老龄化低、老龄化中、老龄化高。
3. 使用 `总人口_线性趋势斜率` 判断趋势，阈值同样为 `> 0.5` 上升、`< -0.5` 下降、其余稳定。

## 8. 缺失值和质量控制

缺失值处理规则：

1. 口粮和饲料粮在 2015-2024 年要求 31 省数据完整，若存在缺失则脚本直接报错。
2. 人口和经济面板数据先按省份和年份排序，对数值字段做省内线性插值。
3. 序列开头和结尾缺失通过双向插值、前向填充和后向填充补齐。
4. 构造特征后，若某个数值特征整列为空则剔除；若仍有少量缺失，则用该特征列中位数补齐。

质量检查规则：

1. 聚类结果必须覆盖 31 个省份。
2. 不允许出现空聚类。
3. 聚类特征不得存在缺失值。
4. `K值评估表.csv` 必须覆盖 K=2-6。
5. 经济聚类不得包含城镇、农村、城市等分城乡字段。

# 四类省份聚类固定 K=4 观察笔记

更新日期：2026-06-05

## 1. 笔记目的

本文不再作为“固定 K=4 调整计划”，而是作为四类省份聚类的细节观察笔记。重点记录：

1. 四类聚类共同使用的方法。
2. 每个聚类任务使用的数据特征。
3. 每个聚类任务的项目内原始数据出处。
4. 原始数据在入模前的字段、年份、省份覆盖、缺失和数值范围情况。
5. 当前固定 `K=4` 输出结果中，每个簇的省份组成和核心中心特征，便于后续观察簇内差异和簇间边界。

本文记录的是项目内脚本和 CSV 可追溯的信息，不额外追溯 CSV 之外的统计年鉴或外部数据库来源。

## 2. 当前四类聚类任务

四类聚类是四个独立任务，均以 31 个省级行政区为样本，不含港澳台。当前四个任务均固定 `K=4`。

| 聚类任务 | 脚本 | 当前 K | 入模省份数 | 主要观察对象 |
| :--- | :--- | ---: | ---: | :--- |
| 口粮消费聚类 | `food_grain_clustering/code/run_food_grain_clustering.py` | 4 | 31 | 省份人均口粮消费水平、变化方向和波动 |
| 饲料粮消费聚类 | `feed_grain_clustering/code/run_feed_grain_clustering.py` | 4 | 31 | 省份人均动物性消费折算饲料粮水平、变化方向和波动 |
| 人口结构聚类 | `population_clustering/code/run_population_clustering.py` | 4 | 31 | 人口规模、城乡结构、年龄结构、家庭规模和户数变化 |
| 经济水平与价格指标聚类 | `economy_clustering/code/run_economy_clustering.py` | 4 | 31 | GDP、收入、消费支出、CPI、食品 CPI、粮食 CPI 的水平、趋势和波动 |

## 3. 通用聚类方法

四类任务都调用 `common/clustering_utils.py` 中的通用函数。当前固定 K=4 的主流程如下：

```text
读取项目内原始 CSV
筛选入模年份和字段
整理为省份-年份长表
必要时补齐年度面板缺失
按省份构造特征表
筛选数值特征并处理特征缺失
对数值特征做 z-score 标准化
在标准化后的完整特征空间中执行 KMeans，固定 K=4
输出聚类结果、聚类中心、标准化特征、质量检查和图表
```

### 3.1 特征构造方法

通用序列特征由 `build_series_features()` 构造。对每个省份、每个入模指标，先按年份排序，再生成以下特征：

| 特征 | 含义 |
| :--- | :--- |
| `均值` | 入模年份内该省该指标的算术平均值 |
| `最新年值` | 入模期最后一年的值，口粮/饲料粮为 2024 年，人口/经济为 2024 年 |
| `首年值` | 入模期第一年的值，口粮/饲料粮为 2015 年，人口/经济为 2000 年 |
| `变化量` | `最新年值 - 首年值` |
| `变化率` | `变化量 / 首年值`，首年值接近 0 时记为缺失，后续在特征处理阶段补齐 |
| `线性趋势斜率` | 用年份和指标值做一元线性拟合得到的斜率 |
| `标准差` | 入模期内该指标的波动幅度，人口聚类不使用 |
| `变异系数` | `标准差 / 均值`，人口聚类不使用 |

口粮、饲料粮、经济聚类使用 8 类序列特征；人口聚类不使用标准差和变异系数，每个指标使用 6 类水平和趋势特征。

### 3.2 缺失处理方法

1. 口粮和饲料粮在 2015-2024 年要求 31 省目标值完整；若筛选后存在缺失，脚本直接报错。
2. 人口和经济先按省份、年份排序，对入模数值字段执行省内线性插值，并用前向、后向填充补齐序列两端缺失。
3. 构造特征后，若某个数值特征整列为空则剔除；若仍存在局部缺失，则用该特征列中位数补齐。
4. 当前输出的四个 `质量检查.csv` 均显示：覆盖 31 省、无空聚类、聚类特征无缺失。

### 3.3 标准化与 KMeans

聚类实际使用标准化后的完整特征表，不使用原始量纲直接计算距离。

| 项目 | 当前设置 |
| :--- | :--- |
| 标准化 | 每个数值特征列减均值、除标准差，标准差小于 `1e-12` 时按 1 处理 |
| 聚类算法 | 项目内自实现 KMeans：`fit_simple_kmeans()` |
| 距离目标 | 最小化标准化特征空间中的类内平方距离和，即 `inertia` |
| `n_clusters` | 固定为 4 |
| `random_state` | 42 |
| `n_init` | 80 次随机初始化，取 `inertia` 最小结果 |
| `max_iter` | 每次初始化最多 300 轮 |
| 空簇处理 | 若某轮出现空簇，用距离最近中心最远的样本重置该簇中心 |

当前固定 K=4 后，`cluster_feature_table(..., default_k=4)` 不再在主流程中生成 `K值评估表.csv`。`evaluate_k_values()` 和 `choose_k()` 仍保留在通用工具中，但本轮四类聚类结果不以自动选 K 为依据。

### 3.4 PCA 图的含义

`PCA二维聚类散点图.png` 只用于二维可视化。PCA 坐标由标准化后的完整特征表通过 SVD 得到，帮助观察样本分布和簇间分离情况，但不参与 KMeans 拟合。

### 3.5 标签生成

口粮、饲料粮和经济聚类使用 `make_cluster_labels()`：

1. 对每个簇计算核心水平指标的簇均值。
2. 用簇均值的 33% 和 67% 分位点划分低水平、中等水平、高水平。
3. 对趋势斜率取簇均值，`> 0.5` 标记为上升型，`< -0.5` 标记为下降型，其余标记为稳定型。
4. 经济聚类把水平标签文字替换为高经济水平、中等经济水平、低经济水平。

人口聚类使用 `apply_population_labels()`：

1. 用 `总人口_最新年值` 的簇均值划分高人口规模、中等人口规模、低人口规模。
2. 用 `65岁及以上占比_最新年值` 的簇均值划分老龄化高、老龄化中、老龄化低。
3. 用 `总人口_线性趋势斜率` 判断上升型、下降型、稳定型。

需要注意：标签是按簇中心分位点自动命名的描述性标签，不等同于严格业务分档。固定 K=4 后，部分任务会出现两个簇标签文字相同但中心值和成员不同的情况，例如饲料粮聚类和经济聚类。


## 10. 后续观察问题清单

1. 单省簇是否合理：口粮的西藏、人口的河南、经济的北京上海双省簇都需要结合原始年度序列确认。
2. 标签相同但簇不同的情况是否需要人工补充命名：饲料粮 `cluster=0/1` 和经济 `cluster=0/1` 都存在相同自动标签。
3. K=4 是否造成过细拆分：应结合 `聚类中心_标准化.csv` 和 PCA 图判断簇间距离是否足够。
4. 人口任务中河南 2000-2001 年年龄结构为 0 的原始值是否应继续保留，还是需要按缺失处理后重跑。
5. 经济任务中价格指数特征对聚类边界的影响是否足够明显，必要时可对价格指数和经济规模特征分组观察中心热图。
6. 口粮和饲料粮都是单指标时间序列特征聚类，观察时应重点区分“水平差异”和“趋势/波动差异”，不要只看最新年值。
